# Тестовое задание - Ордынцев Виктор

## Работа с документацией и ИИ

При разработке решения использовалась официальная документация Ozon Seller API, а также AI-инструменты для помощи в написании и проверке кода.

### Как работали с документацией

Перед началом разработки была изучена документация Ozon API:
- описание методов API;
- необходимые HTTP-запросы;
- параметры запросов;
- формат входных и выходных данных;
- ограничения API (лимиты запросов, пагинация, максимальное количество элементов).

Для каждого используемого метода сначала проверялось:
- существует ли такой endpoint в официальной документации;
- какие параметры обязательные;
- какой формат ответа возвращается.

Использованные методы:
- получение списка товаров;
- получение информации о товарах;

### Использование ИИ
При выполнение задания использовались ИИ модели по большей части для поиска ошибок или неточностей. Также ИИ помогала быстро разобраться в документации.

Если кратко, модель использовалась для:
- генерации структуры кода;
- поиска ошибок;
- объяснения документации;
- улучшения обработки данных.

В модель передавались небольшие кусочки кода или документации, появснение ожидаемого результата или то проблемы с которой столнулся и не вижу решения.

### Проверка корректности ответов ИИ

Чтобы убедиться, что модель не выдумывает методы API и не использует устаревшие данные:

1. Проверялось в официальной документации Ozon API.
2. Сверялись:
   - название метода;
   - HTTP-метод (GET/POST);
   - URL endpoint;
   - параметры запроса;
   - структура ответа.
3. Проверялись реальные ответы API через тестовые запросы.


Но в данной работе ИИ использовалось не особо много, так как сами задачи не высокой сложности и многие вопросы решались обычным "гугулением"

# Задача 3

## Найденные крайние случаи

В процессе обработки заметил несколько крайних случаев:

* У некоторых товаров не был указан бренд отдельно, поэтому пытался определить его из названия. Если определить было невозможно — оставлял «Без бренда».
* Бренды могли быть записаны по-разному (Mavico, MAVICO), поэтому приводил их к одному виду.
* OEM-номера встречались в разных форматах, поэтому часть значений приходилось определять по шаблонам.
* Количество товара иногда было указано в названии (2 шт, пара), а иногда отсутствовало — в таком случае ставил значение 1.
* Были разные форматы цены и пустые значения, которые требовали нормализации.

Самыми неоднозначными были определение бренда и OEM-номера, так как эти данные иногда приходилось извлекать из названия товара.

# Очистка каталога товаров

## Описание
Скрипт выполняет обработку исходного каталога товаров:
- очистка и нормализация данных;
- определение бренда из названия товара;
- извлечение OEM-номера;
- нормализация цены;
- определение количества товара;
- удаление дубликатов.
